# 图片优化（统一走 Next Image 链路）

## 上下文

- 部署：仅 Vercel
- 目标：
  - 全站“栅格图”统一走 `next/image` 优化链路（避免第三方组件渲染 `<img>` 绕过 `/_next/image`）
  - `fill` / 响应式图片必须提供 `sizes`，减少过度下载，提升 LCP 可能性
- 约束：
  - 不改 `next.config.mjs` 的 `images.remotePatterns`（保持现状）
  - 不处理 MDX 图片；不存在用户上传/任意外链图片
  - 代码里仍可能出现 remote svg/logo、AI 结果 URL：保证类型与运行不报错
- 额外：包含 favicon 替换（避免下载超大 favicon）

## 计划

1. 更新性能自检清单（图片必须走 `next/image`，`fill`/响应式必须有 `sizes`）
2. 将 `LazyImage` 从 `react-lazy-load-image-component` 迁移到 `next/image`（并用 TS 类型强制 `fill+sizes` 或 `width+height`）
3. 修复所有 `LazyImage` 使用点，补齐尺寸与合理 `sizes`
4. 为关键响应式/`fill` 的 `<Image>` 补齐 `sizes`
5. 修复明显“展示尺寸 << 请求尺寸”的错配（小头像/小 logo）
6. 替换 favicon：生成小尺寸 `favicon.ico`，并采用 Next 文件式 metadata（移除手写 `<link rel="icon">`）
7. 移除不再需要的依赖并更新锁文件
8. 验证：`pnpm lint`、`pnpm build`，并做关键规则扫描
