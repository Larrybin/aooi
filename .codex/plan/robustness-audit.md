[任务] 代码健壮性审计（仅 src/，输出报告+整改清单）

## 背景与目标
- 目标：对仓库 `src/` 进行“代码健壮性”审计，输出可直接落地的整改清单。
- 交付物：
  - 审计报告（风险矩阵：严重度×影响面×触发概率）
  - 整改清单（每条含文件/位置/问题/建议方案/回归点/工作量预估/优先级）
- 约束：不改代码，仅审计与建议；允许提出可能影响行为的改动建议。

## 范围
- 仅包含：`src/**`
- 不包含：`scripts/`、配置文件、CI、依赖升级（可作为建议但不在本轮落地）

## 审计维度（最小基线）
1. 输入与数据边界：`JSON.parse`/外部回包/DB 字段/表单数据的容错与校验（优先 Zod/安全解析）
2. 错误处理一致性：Route Handler 是否统一使用 `withApi()` + `ApiError`（避免不可控异常直接冒泡）
3. UI 兜底：App Router 的 `error.tsx`/`global-error.tsx`/`not-found.tsx` 覆盖情况（分段与全局）
4. 可观测性：请求级上下文（`requestId/route/method`）是否贯穿日志与错误响应
5. 类型逃逸控制：`@ts-ignore`、`as any`、`as unknown as` 的原因与替代方案（类型收敛/运行时守卫）
6. 外部调用健壮性：`fetch` 超时、取消、重试/退避、错误消息归一化（按场景）

## 执行步骤
1. 扫描并分类高风险模式（仅 `src/`）
2. 深读关键链路（AI 任务/支付/聊天/API 包装层）
3. 生成风险矩阵与整改清单（P0/P1/P2）
4. 给出验证与回归建议（最小可行验证点）
