# 任务：默认语言全站无前缀可访问（非默认语言保留前缀，URL 不变的内部 rewrite）

## 背景与现状（基于仓库事实）

- 路由结构全部在 `src/app/[locale]/...`，因此文件系统路由只能命中 `/:locale/...`（如 `/en/pricing`、`/zh/pricing`）。
- 当前存在大量无前缀硬编码链接/配置（例如 `src/config/locale/messages/*/landing.json` 中的 `"/pricing" "/docs" "/blog"` 等），导致在 `/:locale` 页面里也可能出现 `/pricing` 这种无前缀跳转。
- `next-intl` 配置为 `localePrefix: 'as-needed'` + `localeDetection: false`：默认语言允许无前缀，但不会“凭空”让 `/pricing` 命中 `[locale]/pricing`。

## 目标（成功标准）

1. **默认语言全站无前缀可访问**：任意页面 `/<path>`（不带 locale）能渲染默认语言内容，且浏览器地址栏保持无前缀 URL（内部 rewrite，不做 redirect）。
2. **非默认语言保留前缀**：任意页面 `/<nonDefaultLocale>/<path>` 正常渲染对应语言内容；从非默认语言页面点击站内链接仍优先保持该语言（生成带前缀 URL）。
3. **不复制整棵路由树**：继续只维护 `src/app/[locale]/...` 一套页面实现。
4. **不回归静态资源/内部端点**：`/_next/*`、静态文件、`/_vercel/*`、`/api/*` 不被 i18n/auth rewrite 误伤。

## 非目标（明确不做）

- 不改变业务文案/信息架构。
- 不一次性修复目前 `/api/config/get-configs` / `/api/auth/get-session` 的 500（独立问题，另开任务）。

## 官方最佳实践依据（Context7 摘要）

- `next-intl`：
  - `localePrefix` 可以改变 URL 是否显示前缀；`localePrefix: 'never'` 通过 **内部 rewrite** 让 `[locale]` 路由仍收到 locale（代价是 alternate links 等能力受限）。
  - 允许在 next-intl middleware 之后追加自定义 `NextResponse.rewrite()`。
  - middleware matcher 推荐排除 `/_next` 与包含 `.` 的静态文件，避免误 rewrite。
- Next.js：
  - middleware/proxy 的 matcher 建议使用 negative matching 排除 `/_next/static`、`/_next/image`、metadata 文件等。
- OpenNext（面向 Cloudflare）：
  - routing layer 会处理 middleware/rewrites/i18n；将“默认无前缀”作为前置层规则比复制两套路由更稳定、更易迁移。

## 方案选择

采用：**在 `middleware.ts` 实现“无前缀 → 内部补默认 locale 前缀”的 rewrite**，并确保站内链接生成在非默认语言下带前缀。

### 路由规则（逻辑定义，不写最终代码）

对任意非 API 请求：

1. 若 pathname 已以支持的 locale 前缀开头（例如 `/zh/...`、`/en/...`）：按现有逻辑继续走 `src/request-proxy.ts`（i18n + 轻鉴权）。
2. 若 pathname **不带 locale 前缀**（例如 `/pricing`、`/settings/billing`、`/admin`、`/`）：
   - 先执行轻鉴权（依据“去 locale 后的 path”判断 `/admin|/settings|/activity`）。
   - 再 `rewrite` 到 `/${defaultLocale}${pathname}`（根路径 `/` rewrite 到 `/${defaultLocale}`）。
   - response 上保持注入 `x-request-id`；request headers 上游也携带 `x-request-id`。

同时通过 matcher 过滤：排除 `/_next/*`、`/api/*`、静态文件（含 `.`）、`/_vercel/*`、metadata/`.well-known` 等。

## 变更计划（原子步骤）

### 1) 路由与 i18n 前置层改造

- 文件：`middleware.ts`
  - 新增：判断是否带 locale 前缀（读取 `locales` 与 `defaultLocale`）。
  - 新增：对无前缀 path 执行内部 rewrite 到 `/${defaultLocale}${pathname}`（URL 不变）。
  - 调整：将 `x-request-id` 注入同时应用到 rewrite/redirect/next/proxy 的所有分支。
  - 保持：`/api` 分支仍只注入 requestId（不做 i18n/auth）。
  - 加固：matcher 进一步排除 metadata 与特殊文件（除现有 `.*\\..*` 外，明确列出 `favicon.ico/robots.txt/sitemap.xml/manifest` 等，确保兼容未来变更）。

- 文件：`src/request-proxy.ts`
  - 保持职责：仅处理“已带 locale 前缀的页面请求”的 next-intl middleware + 轻鉴权。
  - 可选小优化（如需要）：抽出“去 locale 后的 path”与“是否需要 session”的判定函数，避免 `middleware.ts` 与 `src/request-proxy.ts` 逻辑漂移。

### 2) 站内链接生成修正（确保非默认语言保留前缀）

目标：在 `/<locale>` 页面内，任何站内跳转（尤其是由 JSON 配置驱动的 header/nav/buttons）应使用 `@/core/i18n/navigation` 导出的 `Link/getPathname`，让 next-intl 依据当前 locale 生成正确前缀。

- 识别入口：
  - 从 `src/config/locale/messages/*/landing.json` 读取的 `url` 字段目前是无前缀字符串。
  - 需要确认渲染这些 url 的组件是否用了 `next/link` 或原生 `<a>`，导致不带 locale 前缀。
- 改造原则：
  - JSON 中继续存“无前缀的语义路径”（如 `/pricing`），不要把 locale 写进内容配置（避免数据与路由策略耦合）。
  - 渲染侧统一用 `@/core/i18n/navigation` 的 `Link` 组件承载跳转，从而在 `zh` 环境下生成 `/zh/pricing`，在默认语言下仍为 `/pricing`（as-needed 语义）。

### 3) 文档同步

- 文件：`docs/ARCHITECTURE_REVIEW.md`
  - 更新“中间件与路由闸门”段落，补充“默认语言无前缀的内部 rewrite 策略”与 matcher 排除清单（强调避免静态资源误拦截）。

## 验证清单（必须全部通过）

### A. 默认语言无前缀（URL 不变）

- `GET http://localhost:3000/`：200，页面内容与 `/en` 一致（但地址栏保持 `/`）。
- `GET /pricing`、`/docs`、`/blog`、`/admin`、`/settings/billing`：不带前缀也能渲染/跳转（需要登录的路由仍按规则拦截）。

### B. 非默认语言保留前缀

- `GET /zh`、`/zh/pricing`：200。
- 在 `/zh` 页面点击 Header 的 Pricing/Docs/Blog：应跳到 `/zh/pricing`、`/zh/docs`、`/zh/blog`（而不是无前缀）。

### C. 排除项不被影响

- `GET /_next/static/...`、`/_next/image?...`：200（不得出现被 rewrite 后的 not-found）。
- `GET /robots.txt`、`/favicon.ico`：200。
- `POST /api/*`：middleware 只注入 request-id，不做 i18n rewrite。

## 风险与缓解

- 风险：内部 rewrite 使“可见 URL”和“内部路由”不同，可能影响某些 metadata route（next-intl 提示：自定义 localePrefix/rewrites 可能影响 OG route 可达性）。
  - 缓解：将 metadata 文件与特殊 route 全部从 matcher 排除；并补充对关键 metadata（icon/robots/sitemap）访问测试。
- 风险：站内仍存在少量 `next/link`/`<a>` 的硬编码跳转，导致在非默认语言下丢失前缀。
  - 缓解：集中改造 header/nav/关键 CTA 渲染组件，后续用 `rg` 做一次全仓扫描并逐步收敛到 `@/core/i18n/navigation`。

## 回滚策略

- 若出现不可预期的路由行为：临时切换为“默认语言 redirect 到 `/${defaultLocale}`”（更显式但不满足 URL 不变），用于快速恢复页面可达性；然后再逐步修复 rewrite 逻辑。
