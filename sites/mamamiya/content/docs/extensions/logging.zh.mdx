本项目在 server 侧使用统一 logger，用于：

- 生产环境默认仅输出 `error`（降低噪音）
- `debug` 仅在 `NEXT_PUBLIC_DEBUG=true` 时启用
- 按 key 名命中对敏感字段统一脱敏
- 通过统一结构化字段提升日志可检索性

## Route Handler 用法

建议在 Route Handler 中优先使用 `getRequestLogger(req)`，一次性绑定请求上下文（`route`/`requestId`/`method`）：

- `src/shared/lib/request-logger.server.ts`

## 结构化字段

通用字段（建议统一）：

| 字段        | 类型      | 说明                                          |
| ----------- | --------- | --------------------------------------------- |
| `route`     | `string`  | 仅记录 pathname（不包含 query）               |
| `requestId` | `string`  | 优先使用 `x-request-id`，否则回退到平台 id    |
| `method`    | `string`  | HTTP 方法                                     |
| `provider`  | `string`  | 支付/邮件等 provider 名称                     |
| `userId`    | `string`  | 敏感标识；仅在必要时记录（优先 debug 或哈希） |
| `error`     | `unknown` | 建议传 `Error`；logger 会序列化并脱敏         |

## 安全约束

- 禁止记录完整 webhook payload、授权头、cookie、原始 token。
- 优先记录摘要字段（id/status/count），避免整对象输出。
- 对客户端返回保持泛化错误信息，详细上下文只进 server 日志。

## 脱敏命中规则（按 key 名）

logger 会对对象进行递归脱敏：当 key 名命中以下规则时，该字段会被替换为 `"[REDACTED]"`。

> 以 `src/shared/lib/logger.server.ts` 为准；此处为当前实现的同步摘要，后续如代码更新请同步更新本文档。

| 规则类型      | 命中示例（key）                                        |
| ------------- | ------------------------------------------------------ |
| 固定 key      | `authorization`、`cookie`、`set-cookie`                |
| password 类   | `password`、`pwd`、`pass`（含子串命中）                |
| secret 类     | `secret`、`client_secret`、`private_key`（含子串命中） |
| token 类      | `token`（含子串命中）                                  |
| api key 类    | `api_key`、`apikey`、`api-key`（含子串命中）           |
| session id 类 | `session*id`（同时包含 `session` 且以 `id` 结尾）      |
