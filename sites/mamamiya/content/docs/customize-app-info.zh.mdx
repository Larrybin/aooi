---
title: 自定义应用信息
description: 在当前仓库里统一自定义品牌信息、SEO 元信息、法律页面与站点资源。
---

当前仓库使用构建期站点 identity 管理应用信息与品牌信息。请在 `sites/<site-key>/site.config.json` 中配置 identity。在当前仓库里，`sites/dev-local/site.config.json` 是本地默认站点，`sites/mamamiya/site.config.json` 是生产站点。运行时 settings 只保留站内开关和第三方集成配置。

## 当前仓库支持哪些“应用信息”自定义

| 项目              | 当前仓库是否支持 | 主要入口                                                                     |
| ----------------- | ---------------- | ---------------------------------------------------------------------------- |
| 应用名称          | 支持             | `sites/<site-key>/site.config.json -> brand.appName`                         |
| 应用域名 / Origin | 支持             | `sites/<site-key>/site.config.json -> brand.appUrl`                          |
| 支持邮箱          | 支持             | `sites/<site-key>/site.config.json -> brand.supportEmail`                    |
| Logo              | 支持             | `sites/<site-key>/site.config.json -> brand.logo` 或 `public/logo.png`       |
| Favicon           | 支持             | `sites/<site-key>/site.config.json -> brand.favicon` 或 `public/favicon.ico` |
| 社交分享预览图    | 支持             | `sites/<site-key>/site.config.json -> brand.previewImage`                    |
| 隐私政策          | 支持             | `sites/<site-key>/content/pages/privacy-policy*.mdx`                         |
| 服务条款          | 支持             | `sites/<site-key>/content/pages/terms-of-service*.mdx`                       |
| Sitemap           | 支持             | `src/app/sitemap.ts`                                                         |
| Robots            | 支持             | `src/app/robots.ts`                                                          |

## 推荐的自定义顺序

## 1. 先配置应用基础信息

配置构建期站点 identity：

```json
{
  "brand": {
    "appName": "YourAppName",
    "appUrl": "https://example.com",
    "supportEmail": "support@example.com",
    "logo": "/logo.png",
    "favicon": "/favicon.ico",
    "previewImage": "/logo.png"
  }
}
```

规则：

- `brand.appUrl` 必须是纯 origin，例如 `https://example.com`
- `brand.logo` / `brand.favicon` / `brand.previewImage` 必须是 public 路径，或被图片策略允许的绝对 URL
- `SITE` 显式选择站点。生产命令必须设置 `SITE=mamamiya`；本地 lint/test 命令在未设置 `SITE` 时只使用内部 `dev-local` 测试站点。
- `NEXT_PUBLIC_APP_URL` 由 `brand.appUrl` 生成给 infra/deploy 使用；它不是 identity 来源，也不是 fallback。

## 2. 再配置运行时 settings

登录后台后进入：

`/admin/settings/general`

品牌 identity 不再从后台 settings 编辑。第三方集成配置仍保存在配置表中。

## 3. 替换站点资源

默认资源文件：

- `public/logo.png`
- `public/favicon.ico`

如果你不想直接覆盖默认文件，而是希望走独立路径，可以：

- 修改当前 `sites/<site-key>/site.config.json` 里的 `brand.logo` / `brand.favicon` / `brand.previewImage`

这些资源会影响：

- docs 顶部导航 Logo
- 登录/注册页 Logo
- 404 页面 Logo
- landing / admin / chat 的品牌 Logo
- 全局 SEO metadata 与 favicon

## 4. 修改法律页面

编辑当前站点的 `sites/<site-key>/content/pages/` 下这些文件，例如：

- `sites/<site-key>/content/pages/privacy-policy.mdx`
- `sites/<site-key>/content/pages/privacy-policy.zh.mdx`
- `sites/<site-key>/content/pages/terms-of-service.mdx`
- `sites/<site-key>/content/pages/terms-of-service.zh.mdx`
- `sites/<site-key>/content/pages/terms-of-service.zh-TW.mdx`

其中类似这些占位内容会在渲染时自动替换：

- `YourAppName`
- `your-domain.com`
- `support@your-domain.com`

## 5. 检查 SEO / 抓取配置

核心文件：

- `src/shared/lib/seo.ts`
- `src/app/sitemap.ts`
- `src/app/robots.ts`

当前默认行为：

- favicon 来自 `site.brand.favicon`
- Open Graph / Twitter 预览图默认来自 `site.brand.previewImage`
- canonical 与 sitemap 使用 `site.brand.appUrl`

## 当前仓库里，这些能力分别在哪里实现

### 环境变量默认值

- `src/config/index.ts`

### 运行时品牌信息解析

- `src/shared/lib/brand-placeholders.server.ts`
- `src/shared/lib/brand-identity.ts`

### Metadata / SEO

- `src/shared/lib/seo.ts`
- `src/app/[locale]/layout.tsx`
- `src/app/sitemap.ts`
- `src/app/robots.ts`

### 会使用品牌资源的页面

- `src/app/[locale]/(docs)/layout.config.tsx`
- `src/app/[locale]/(auth)/layout.tsx`
- `src/app/not-found.tsx`

## 和 ShipAny 那种文档相比，目前差异在哪里

核心能力已经基本对齐：

- **应用名 / 域名 / 支持邮箱**：支持后台配置
- **Logo / Favicon / 预览图**：支持后台上传，也支持 env / `public/` 兜底

当前唯一保留的简单约束是：上传依赖已配置的对象存储，而不是把文件直接存进数据库。
